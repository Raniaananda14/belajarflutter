01
Arsitektur & Landasan Sistem

Deskripsi Teknis

Detail tumpukan teknologi (tech stack) dan implementasi logika inti BizGrow untuk menjamin fungsionalitas luring (offline) yang optimal dan performa tinggi.

🖥️ UI & Framework Layer

Membangun antarmuka modern yang reaktif menggunakan Flutter SDK dengan struktur Stateful/Stateless widget modular dan sistem tema adaptif (Context Extensions).

💾 Data Persistence Layer

Penyimpanan data relasional lokal yang efisien menggunakan SQLite (via database helper terpusat). Mendukung migrasi skema dinamis dan kueri relasional cepat.

🗺️ Custom Painting Layer

Menggambar wilayah peta interaktif tanpa ketergantungan API eksternal menggunakan CustomPainter dan Canvas API. Menyajikan bayangan elevasi (drop shadow) 3D dan grid koordinat presisi.

🔒 Security & Session Management

Manajemen sesi pengguna berbasis waktu, hashing sederhana, serta pemulihan kredensial via mekanisme Security Questions di halaman login.

04
Iterasi Berdasarkan Masukan

User Testing & Feedback

Bagaimana tim BizGrow menanggapi dan mengimplementasikan masukan dari pengguna selama fase pengujian prototype.

⚠️

Form Informasi Pribadi (Input NIK)

"Pengguna merasa keberatan memasukkan NIK asli saat mendaftar karena masalah privasi data pribadi."

✓ Solusi: Mengubah label input NIK menjadi "Nomor Telepon" dan memetakan datanya ke dalam kolom database 'nik' agar kompatibilitas data tetap terjaga.

🛑

Unggah Bukti Pembayaran

"Setelah memilih gambar bukti pembayaran, sistem langsung melakukan checkout otomatis secara tiba-tiba."

✓ Solusi: Memisahkan logika pengunggahan. Unggahan hanya memicu snackbar notifikasi sukses, pengguna harus menekan tombol "Checkout Sekarang" secara manual.

📐

Invoice A4 Overflow saat Multi-Product

"Ketika membeli lebih dari 2 produk, catatan invoice dan daftar produk bertabrakan dan menghasilkan error overflow."

✓ Solusi: Membungkus daftar belanjaan dan catatan tambahan dalam layout scrollable dinamis dengan tinggi fleksibel.

05
Evolusi Pengembangan

Perubahan dari Proposal (Pivot)

Penyesuaian arah pengembangan dari desain awal guna meningkatkan efektivitas bisnis dan pengalaman pengguna (UX).

1. Pembagian Fitur Pengaturan Berdasarkan Role

Pada proposal awal, pengaturan profil bersifat generik. Kami memisahkannya: Owner mendapatkan setelan Penjualan & Target; Pembeli mendapatkan setelan Status Pesanan.

2. Selector Metode Pembayaran Interaktif

Mengubah kolom input teks bank manual menjadi grid selector visual yang interaktif (Mandiri, BCA, BRI, QRIS, OVO, Tunai) dengan instruksi transfer yang dinamis.

3. Akses Invoice Mandiri dari Tracking Order

Menambahkan tombol "Lihat Invoice" (Read-Only) pada daftar pesanan selesai di menu "Pesanan Saya" untuk memudahkan pencatatan tanpa harus masuk ke alur transaksi baru.

💡 Mengapa Kami Melakukan Pivot?

Pivot ini tidak mengubah tujuan inti aplikasi, melainkan:

  • Meningkatkan **keamanan data pengguna** (Penghapusan NIK wajib).
  • Memastikan **akurasi invoice** (Pemisahan tombol checkout & bukti).
  • Meningkatkan **usabilitas visual** (Grid bank & visualisasi peta 3D).
06
Kesimpulan & Langkah Berikutnya

Output MVP & Langkah Ideation

BizGrow siap dikembangkan lebih lanjut menuju fase produksi penuh berdasarkan landasan kokoh prototype saat ini.

📦 Status MVP

Kode bersih dan modular bebas error kompilasi. Semua fungsionalitas utama yang diuji luring (offline) berjalan dengan performa rendering 60 FPS pada emulator Android/iOS.

⚡ Rencana Optimasi

Menambahkan integrasi REST API atau Firebase sync untuk menyinkronkan database SQLite lokal secara berkala saat perangkat terhubung ke internet (online-first sync).

🚀 Fitur Ideation Baru

Menerapkan kecerdasan buatan sederhana untuk memprediksi tren penjualan mingguan langsung di dashboard Owner berdasarkan riwayat transaksi lokal.
